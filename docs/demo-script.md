# 100-second demo script

## 0–12 seconds — the fear

Show the old returns console and one sentence: “The workflow runs the business, but its rules were never documented.” Process a damaged return once.

## 12–30 seconds — evidence, not a transcript

TraceForge records the request, response, and inventory before/after state. Open one rule: “Refunded damage never returns to sellable stock,” with evidence IDs and confidence.

## 30–48 seconds — bounded contract

Show six inferred rules plus one visible unknown. A counterexample scenario resolves the minimum ambiguity. Say “TraceForge names what it does not know.”

## 48–66 seconds — candidate build

Show the isolated candidate diff created by Codex. Avoid agent avatars; show tool status, files changed, tests added, and elapsed time.

## 66–84 seconds — rejection

Run the original and replacement together. The replacement incorrectly restores damaged inventory. The proof ledger turns orange and links directly to the mismatched entity snapshots.

## 84–96 seconds — repair and independent rerun

Codex patches only the replacement. The verifier resets both systems and runs again. The same evidence row turns green with a new run ID; the earlier failed run remains available.

## 96–100 seconds — handoff

End on the replacement, source diff, and proof bundle: “Show the workflow. Ship the replacement—with evidence.”
