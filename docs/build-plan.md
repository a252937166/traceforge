# Build Week plan

## First 48 hours — stop-loss gate

The project continues only if this exact loop works from a clean checkout:

1. Execute one legacy workflow.
2. Store evidence for an API result and a database transition.
3. Express one rule in the behavior contract.
4. Run the same scenario against the replacement.
5. Reject one real mismatch.
6. Apply a candidate patch and pass an external rerun.

If this is unstable by the end of July 15 Beijing time, narrow to accessibility repair while retaining the capture, candidate, and proof engine.

## Days 3–4

- Six to eight rules and eight deterministic scenarios.
- Evidence drill-down and a downloadable proof bundle.
- A single isolated Codex writer integration.
- Ten clean runs with failure evidence preserved.

## Days 5–6

- GPT-5.6 read-only archaeology behind structured outputs.
- Counterexample generation and uncertainty UI.
- Cost, latency, and human-intervention metrics.
- Security/redaction audit.

## Days 7–8

- Deploy a reproducible public demo.
- Record a 100-second primary demo and a longer technical walkthrough if allowed.
- Finalize README, architecture, build log, repository license, and submission text.
- Re-check official eligibility, tracks, model requirements, and video/repository rules after publication.
